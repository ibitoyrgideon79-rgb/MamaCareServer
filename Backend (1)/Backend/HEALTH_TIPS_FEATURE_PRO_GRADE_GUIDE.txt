
MAMACARE BACKEND - HEALTH TIPS FEATURE PRO-GRADE CODE GUIDE
===========================================================

Purpose of this document
------------------------
This guide explains the Health Tips work from the code point of view, not only from the product/feature point of view. In this backend, the tips are currently implemented as medication safety tips that are returned together with the user's medication home screen data.

Feature in one sentence
-----------------------
When an authenticated user calls the medication home endpoint, the backend returns two pieces of data in one response: the user's medications for today and a small list of educational health tips.

Main endpoint
-------------
GET /api/v1/medications/today

Controller method:
- File: src/main/java/com/Mamacare/Backend/MedicationRemiderPackage/Controller/MedicationController.java
- Method: getTodayMedications(Authentication authentication)

Response DTO:
- File: src/main/java/com/Mamacare/Backend/MedicationRemiderPackage/DTOs/MedicationHomeResponse.java
- JSON fields:
  - today_medications: list of today's medication cards
  - tips: list of health/medication tip cards

High-level request flow
-----------------------
1. The client sends GET /api/v1/medications/today with a valid JWT/access token.
2. Spring Security resolves the logged-in user and passes the Authentication object into the controller method.
3. MedicationController delegates the work to MedicationService.getTodayMedications(authentication).
4. MedicationService extracts the current user from Authentication.
5. MedicationService loads all active medications for that user from MedicationRepo.
6. MedicationService filters those medications using MedicationScheduleCalculator.shouldShowToday(...), so only medications due today remain.
7. MedicationService checks MedicationIntakeLogRepo to know which of today's medications were already marked TAKEN.
8. MedicationService maps the Medication entities into MedicationResponse DTOs.
9. MedicationService asks MedicationTipService.getTips() for the educational tips.
10. MedicationService builds MedicationHomeResponse with both todayMedications and tips.
11. The controller returns HTTP 200 OK with the combined JSON payload.

Important code pieces
---------------------

1. MedicationController
~~~~~~~~~~~~~~~~~~~~~~~
The controller is the HTTP entry point. It does not contain business rules. Its job is to expose REST routes and call the service layer.

Relevant code shape:

    @GetMapping("/today")
    public ResponseEntity<MedicationHomeResponse> getTodayMedications(Authentication authentication) {
        return ResponseEntity.ok(medicationService.getTodayMedications(authentication));
    }

Why this matters:
- The route is easy for the frontend to consume: /api/v1/medications/today.
- The controller stays thin, which is good Spring Boot design.
- The Authentication object is forwarded to the service, so user-specific data is handled below the controller layer.

2. MedicationService.getTodayMedications(...)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
This is the main orchestration method for the home-screen data.

Relevant code shape:

    @Transactional(readOnly = true)
    public MedicationHomeResponse getTodayMedications(Authentication authentication) {
        User currentUser = getCurrentUser(authentication);
        List<Medication> activeMedications = medicationRepository
                .findByUserIdAndActiveTrueOrderByMedicationTimeAsc(currentUser.getId());
        List<Medication> todayMedications = activeMedications.stream()
                .filter(scheduleCalculator::shouldShowToday)
                .toList();
        Set<Long> takenMedicationIds = loadTakenMedicationIdsForToday(todayMedications);

        List<MedicationResponse> medicationResponses = todayMedications.stream()
                .map(medication -> toResponse(medication, takenMedicationIds))
                .toList();

        return MedicationHomeResponse.builder()
                .todayMedications(medicationResponses)
                .tips(medicationTipService.getTips())
                .build();
    }

What each part does:
- @Transactional(readOnly = true): tells Spring this method only reads data. That is cleaner and safer than opening a write transaction.
- getCurrentUser(authentication): validates that the request is authenticated and extracts the logged-in User object.
- medicationRepository.findByUserIdAndActiveTrueOrderByMedicationTimeAsc(...): loads only this user's active medications, ordered by time.
- scheduleCalculator.shouldShowToday: applies the schedule/frequency rules, so the response does not show medications that are not due today.
- loadTakenMedicationIdsForToday: checks intake logs and returns the IDs of medications already taken today.
- toResponse(...): converts database entities into API-safe DTOs.
- medicationTipService.getTips(): injects the health tip cards into the same response.

Why this matters:
- The tips are not an isolated endpoint in the current design. They are part of the medication home response.
- The feature is connected to real user state because the same response includes today's medications and taken/not-taken status.
- The service is responsible for combining dynamic user data with static educational content.

3. MedicationTipService
~~~~~~~~~~~~~~~~~~~~~~~
This service owns the actual tip content.

File:
- src/main/java/com/Mamacare/Backend/MedicationRemiderPackage/Service/MedicationTipService.java

Relevant code shape:

    @Service
    public class MedicationTipService {

        public List<MedicationTipResponse> getTips() {
            return List.of(
                    MedicationTipResponse.builder()
                            .title("Take with water")
                            .body("Water helps many medicines move through the body safely.")
                            .build(),
                    MedicationTipResponse.builder()
                            .title("Keep a steady time")
                            .body("Taking medicine around the same time each day helps build a routine.")
                            .build(),
                    MedicationTipResponse.builder()
                            .title("Ask before changing dose")
                            .body("Talk to your doctor or pharmacist before changing how much you take.")
                            .build()
            );
        }
    }

What this class does:
- It is a Spring @Service, so Spring creates and injects it automatically.
- It returns a List<MedicationTipResponse>.
- It currently stores the tips in code as static content.
- Each tip has a title and body.

Why this design is useful:
- Tip content is separated from MedicationService.
- MedicationService does not need to know the exact tip texts.
- If the project later moves tips into a database, API, or CMS, this service is the natural place to change the implementation without rewriting the controller.

Current limitation:
- The tips are hardcoded. That is simple and reliable for an MVP, but it means changing tips requires a code change and redeploy.

Future upgrade path:
- Add a HealthTip entity/table.
- Add fields like id, title, body, category, pregnancyWeekStart, pregnancyWeekEnd, active, priority, createdAt, updatedAt.
- Replace the hardcoded List.of(...) with a repository query.
- Add admin endpoints to create, update, activate, and deactivate tips.

How the code connects to the database
-------------------------------------
The tips themselves do not currently come from the database. The medication list and taken status do.

Database-backed parts:
- MedicationRepo loads active medications for the logged-in user.
- MedicationIntakeLogRepo checks which medications were taken today.
- MedicationReminderRepo is used elsewhere for reminder scheduling and response mapping.

Non-database-backed part:
- MedicationTipService returns hardcoded tip cards.

This is why the feature is mixed:
- The medication home data is dynamic and user-specific.
- The tips are static and shared by all users.

Authentication and user ownership
---------------------------------
The feature is protected by Spring Security because the controller receives an Authentication object. MedicationService then calls getCurrentUser(authentication).

Important behavior:
- If Authentication is missing or the principal is not a User, the service throws an IllegalArgumentException.
- Repository queries use currentUser.getId(), so one user cannot see another user's medications.
- Tips are not user-specific yet, but they are delivered inside an authenticated response.

Why the implementation is clean
-------------------------------
The feature follows common Spring Boot layering:

Controller layer:
- Exposes HTTP endpoints.
- Does not contain business logic.

Service layer:
- Coordinates authentication, repositories, scheduling rules, mapping, and tips.

DTO layer:
- Defines exactly what JSON the API returns.

Repository layer:
- Handles database queries for medications and intake logs.

Tip service:
- Isolates educational content from the rest of the medication logic.

Frontend integration notes
--------------------------
The frontend should call:

    GET /api/v1/medications/today

Then render:
- response.today_medications as the medication cards/checklist.
- response.tips as the health tip cards.

Recommended UI behavior:
- Show tips even when today_medications is empty.
- Do not treat tips as medical diagnosis.
- Display the body text exactly as informational guidance.
- Add a disclaimer if the design requires it, for example: "Always follow advice from your doctor or pharmacist."

Possible pro-grade improvements

Possible fields:

- title

~~~~~~~~~~~~~~~
Add language support for tip content.

~~~~~~~~~~~~~~~~~~~
Add secured admin endpoints:

6. Medical review workflow
~~~~~~~~~~~~~~~~~~~~~~~~~~
Because the app is health-related, future tip content should ideally have review status.

Possible fields:
- reviewStatus
- reviewedBy
- reviewedAt
- sourceReference

Summary
-------
The Health Tips feature is implemented as a small but well-layered part of the medication home endpoint. The controller exposes GET /api/v1/medications/today, MedicationService builds the authenticated user-specific medication home response, MedicationTipService provides the educational tip cards, and MedicationHomeResponse defines the final JSON shape. The feature is currently static-content based, but the service separation makes it easy to upgrade later into database-driven, personalized, localized, or admin-managed health tips.

package com.Mamacare.Backend.HealthTipsFeature.Services;

import com.Mamacare.Backend.HealthTipsFeature.Dtos.HealthTipAudioGuideResponse;
import com.Mamacare.Backend.HealthTipsFeature.Dtos.HealthTipCardResponse;
import com.Mamacare.Backend.HealthTipsFeature.Dtos.HealthTipHomeResponse;
import com.Mamacare.Backend.HealthTipsFeature.Entity.HealthTip;
import com.Mamacare.Backend.HealthTipsFeature.Entity.HealthTipAudioGuide;
import com.Mamacare.Backend.HealthTipsFeature.Enums.HealthTipCategory;
import com.Mamacare.Backend.HealthTipsFeature.Repo.HealthTipAudioGuideRepo;
import com.Mamacare.Backend.HealthTipsFeature.Repo.HealthTipRepo;
import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.util.Comparator;
import java.util.List;
import java.util.Locale;

@Service
@RequiredArgsConstructor
public class HealthTipsService {

    private final HealthTipRepo healthTipRepository;
    private final HealthTipAudioGuideRepo audioGuideRepository;

    @Transactional(readOnly = true)
    public HealthTipHomeResponse getHome() {
        List<HealthTipCardResponse> tips = healthTipRepository
                .findByActiveTrueAndCategoryNotOrderBySortOrderAsc(HealthTipCategory.DAILY_TIP)
                .stream()
                .map(this::toTipResponse)
                .toList();

        HealthTipCardResponse tipOfTheDay = healthTipRepository
                .findFirstByActiveTrueAndCategoryOrderBySortOrderAsc(HealthTipCategory.DAILY_TIP)
                .map(this::toTipResponse)
                .orElse(defaultTipOfTheDay());

        return HealthTipHomeResponse.builder()
                .heading("Health Tips")
                .subtitle("Small steps, healthy you, healthy baby.")
                .audioGuides(audioGuides())
                .tips(tips.isEmpty() ? defaultTips() : tips)
                .tipOfTheDay(tipOfTheDay)
                .build();
    }

    private List<HealthTipAudioGuideResponse> audioGuides() {
        List<HealthTipAudioGuideResponse> savedGuides = audioGuideRepository
                .findByActiveTrueOrderBySortOrderAsc()
                .stream()
                .map(this::toAudioResponse)
                .sorted(Comparator.comparing(HealthTipAudioGuideResponse::getLanguage))
                .toList();

        return savedGuides.isEmpty() ? defaultAudioGuides() : savedGuides;
    }

    private HealthTipCardResponse toTipResponse(HealthTip tip) {
        return HealthTipCardResponse.builder()
                .id(tip.getId())
                .category(tip.getCategory())
                .title(tip.getTitle())
                .body(tip.getBody())
                .sortOrder(tip.getSortOrder())
                .build();
    }

    private HealthTipAudioGuideResponse toAudioResponse(HealthTipAudioGuide audioGuide) {
        return HealthTipAudioGuideResponse.builder()
                .id(audioGuide.getId())
                .language(audioGuide.getLanguage())
                .title(audioGuide.getTitle())
                .audioUrl(audioGuide.getAudioUrl())
                .durationSeconds(audioGuide.getDurationSeconds())
                .displayDuration(displayDuration(audioGuide.getDurationSeconds()))
                .build();
    }

    private String displayDuration(int durationSeconds) {
        int minutes = durationSeconds / 60;
        int seconds = durationSeconds % 60;
        return String.format(Locale.ENGLISH, "%d.%02d min", minutes, seconds);
    }

    private List<HealthTipAudioGuideResponse> defaultAudioGuides() {
        return List.of(
                HealthTipAudioGuideResponse.builder()
                        .language(com.Mamacare.Backend.HealthTipsFeature.Enums.HealthTipLanguage.YORUBA)
                        .title("Listen in yoruba")
                        .audioUrl("/audio/health-tips-yoruba.mp3")
                        .durationSeconds(75)
                        .displayDuration("1.15 min")
                        .build()
        );
    }

    private List<HealthTipCardResponse> defaultTips() {
        return List.of(
                defaultTip(HealthTipCategory.NUTRITION, "Nutrition", "Eat healthy for you and your baby.", 1),
                defaultTip(HealthTipCategory.HYDRATION, "Stay Hydrated", "Drink enough water daily.", 2),
                defaultTip(HealthTipCategory.REST, "Get Enough Rest", "Good rest supports a healthy pregnancy.", 3),
                defaultTip(HealthTipCategory.STRESS_MANAGEMENT, "Manage Stress", "Stay calm and think positive.", 4)
        );
    }

    private HealthTipCardResponse defaultTipOfTheDay() {
        return defaultTip(
                HealthTipCategory.DAILY_TIP,
                "Tips of the Day",
                "Eat variety of fruits and vegetables everyday.",
                5
        );
    }

    private HealthTipCardResponse defaultTip(
            HealthTipCategory category,
            String title,
            String body,
            int sortOrder
    ) {
        return HealthTipCardResponse.builder()
                .category(category)
                .title(title)
                .body(body)
                .sortOrder(sortOrder)
                .build();
    }
}

Explanation:
The service returns saved database content when available, but falls back to the exact screen copy you provided. This means the feature works on day one and can later become admin-editable.

Important correction:
The screen says "1.25 min". If that means 1 minute 25 seconds, use durationSeconds = 85 and displayDuration = "1.25 min". If it means 75 seconds, display it as "1.15 min". Pick one format and stay consistent. I recommend "1:25" for frontend display because users understand time better that way.

import java.util.List;
import java.util.Optional;

import static org.assertj.core.api.Assertions.assertThat;
import static org.mockito.Mockito.when;

@ExtendWith(MockitoExtension.class)
class HealthTipsServiceTest {

    private HealthTipsService healthTipsService;

    @Mock
    private HealthTipRepo healthTipRepository;

    @Mock
    private HealthTipAudioGuideRepo audioGuideRepository;

    @BeforeEach
    void setUp() {
        healthTipsService = new HealthTipsService(healthTipRepository, audioGuideRepository);
    }

    @Test
    void getHomeReturnsSavedTipsAndAudioGuide() {
        HealthTip nutrition = HealthTip.builder()
                .id(1L)
                .category(HealthTipCategory.NUTRITION)
                .title("Nutrition")
                .body("Eat healthy for you and your baby.")
                .sortOrder(1)
                .active(true)
                .build();
        HealthTip dailyTip = HealthTip.builder()
                .id(2L)
                .category(HealthTipCategory.DAILY_TIP)
                .title("Tips of the Day")
                .body("Eat variety of fruits and vegetables everyday.")
                .sortOrder(5)
                .active(true)
                .build();
        HealthTipAudioGuide audioGuide = HealthTipAudioGuide.builder()
                .id(3L)
                .language(HealthTipLanguage.YORUBA)
                .title("Listen in yoruba")
                .audioUrl("/audio/health-tips-yoruba.mp3")
                .durationSeconds(85)
                .sortOrder(1)
                .active(true)
                .build();

        when(healthTipRepository.findByActiveTrueAndCategoryNotOrderBySortOrderAsc(HealthTipCategory.DAILY_TIP))
                .thenReturn(List.of(nutrition));
        when(healthTipRepository.findFirstByActiveTrueAndCategoryOrderBySortOrderAsc(HealthTipCategory.DAILY_TIP))
                .thenReturn(Optional.of(dailyTip));
        when(audioGuideRepository.findByActiveTrueOrderBySortOrderAsc())
                .thenReturn(List.of(audioGuide));

        HealthTipHomeResponse response = healthTipsService.getHome();

        assertThat(response.getHeading()).isEqualTo("Health Tips");
        assertThat(response.getSubtitle()).isEqualTo("Small steps, healthy you, healthy baby.");
        assertThat(response.getTips()).hasSize(1);
        assertThat(response.getTips().get(0).getTitle()).isEqualTo("Nutrition");
        assertThat(response.getTipOfTheDay().getTitle()).isEqualTo("Tips of the Day");
        assertThat(response.getAudioGuides().get(0).getLanguage()).isEqualTo(HealthTipLanguage.YORUBA);
        assertThat(response.getAudioGuides().get(0).getDisplayDuration()).isEqualTo("1.25 min");
    }

    @Test
    void getHomeUsesDefaultsWhenDatabaseHasNoContent() {
        when(healthTipRepository.findByActiveTrueAndCategoryNotOrderBySortOrderAsc(HealthTipCategory.DAILY_TIP))
                .thenReturn(List.of());
        when(healthTipRepository.findFirstByActiveTrueAndCategoryOrderBySortOrderAsc(HealthTipCategory.DAILY_TIP))
                .thenReturn(Optional.empty());
        when(audioGuideRepository.findByActiveTrueOrderBySortOrderAsc())
                .thenReturn(List.of());

        HealthTipHomeResponse response = healthTipsService.getHome();

        assertThat(response.getTips()).hasSize(4);
        assertThat(response.getTips().get(1).getTitle()).isEqualTo("Stay Hydrated");
        assertThat(response.getTipOfTheDay().getBody())
                .isEqualTo("Eat variety of fruits and vegetables everyday.");
        assertThat(response.getAudioGuides()).hasSize(1);
    }
}

Explanation:
This is a pure service unit test. It mocks repositories, so it does not need PostgreSQL and it will not fail because of database credentials.
